Microsoft nahihikayat ang mga developer gamit ang AI na sumusulat ng malaking bahagi ng code.

Inilalagay ng Microsoft ang generative AI sa gitna ng kanilang mga operasyon sa pag-unlad at mga komersyal na produkto, at mabilis na nakakakuha ng mga gumagamit para sa mga tool na ito. Sinabi ni CEO Satya Nadella na 20 hanggang 30 porsyento ng internal code ng Microsoft ay kasalukuyang isinusulat ng AI, habang ang GitHub Copilot ay umabot sa 400,000 subscribers isang buwan matapos ang paglunsad at higit pa rito ay lampas na sa 1 milyong gumagamit. Ang mga numerong ito ay nagpapakita ng makabuluhang pagbabago sa workflows ng software.

Ang pagkakasunod-sunod ng mga kaganapan ay nagsimula sa mabilis na pagdami ng subscription ng Copilot, na nagpakita ng matibay na demand para sa mga tagagawa ng code. Mula rito, pinalawak ng Microsoft ang kanilang pangako sa pamamagitan ng multi-bilyong dolyar na pamumuhunan sa OpenAI, at isinama ang mga teknolohiya ng OpenAI sa Azure sa pamamagitan ng Azure OpenAI Service. Sa 2025 Microsoft Build conference, ipinakilala ng kumpanya ang mga autonomous AI agents at mga intelligent automation tools na dinisenyo upang pahabain ang tulong ng AI sa buong lifecycle ng software.

Binibigyang-diin ng mga detalye ng operasyon hindi lamang ang kakayahan kundi pati na rin ang mga limitasyon. Ipinahayag ng Microsoft na kayang lumikha ng generative AI ng malaking bahagi ng trabaho—napagmasdan ang Copilot na nagmumungkahi o sumusulat ng hanggang 40 porsyento ng isang file sa ilang pagkakataon. Binanggit ni Nadella na nag-iiba ang performance batay sa wika, na may mas magagandang resulta sa Python at hindi gaanong pare-pareho sa mga wika tulad ng C++. Napansin din ng mga tagamasid sa industriya ang patuloy na debate hinggil sa mga pamantayan para masukat kung ano ang itinuturing na “AI-written” code.

Ang pattern ng mga aksyon ay nagpapahiwatig ng isang maingat, multi-faceted na estratehiya. Pinagsasama ng Microsoft ang internal deployment, rollout ng komersyal na produkto, cloud integration, at pamumuhunan ng kapital upang pabilisin ang pag-aampon. Ang internal na paggamit ay nagpapatunay sa teknolohiya sa malaking sukat, ang mga subscription ng produkto ay kumukuha ng demand mula sa mga developer, ang Azure integration ay nagbibigay ng distribusyon sa enterprise, at ang financing ng OpenAI ay nag-secure ng access sa proprietary model. Sama-sama, ang mga hakbang na ito ay nagpapadali sa mga organisasyon na nais magpatupad ng generative AI sa mga kapaligiran ng pag-unlad.

Sa estruktural na aspeto, binabago nito ang sentro ng pagbabago sa pagiging produktibo mula sa mga nalalabing tool patungo sa mga kakayahan sa antas ng platform. Ang mga autonomous agents at automation na inihayag sa Build ay hindi lamang nakatuon sa coding kundi pati na rin sa testing at deployment workflows, na nagpapahiwatig ng intensyon ng Microsoft na ilubog ang AI sa mga end-to-end na proseso. Sa pamamagitan ng pagtataguyod ng “age of AI agents,” pinaparamdam ng mga lider ng kumpanya na ang AI ay isang patuloy na katuwang at hindi lamang isang pansamantalang tagapagtulong, na binabago ang mga inaasahan kung paano ang mga development teams ay nag-aallocate ng mga gawain at oras.

Ngunit may mga paghihirap at kawalang-katiyakan pa. Ang mga pamantayan sa pagsukat para sa AI-generated code ay hindi pa malinaw, na nagpapahirap sa mga pahayag tungkol sa bahagi ng trabaho na isinagawa ng mga modelo. Ang variability ng performance sa iba’t ibang programming languages ay nagpapahirap para sa pangkalahatang aplikasyon; ang mas magagandang resulta sa mga wika tulad ng Python ay hindi nangangahulugan ng katulad na tagumpay sa C++ o iba pang ecosystem. Ang mga limitasyong ito ang magpapasya kung saan at paano ilalagay ng mga organisasyon ang mga generative coding tools sa malapit na hinaharap.

Ang pananaw ay nag-uugnay sa mabilis na pag-aampon at mga kondisyunal na limitasyon. Ang mga integrasyon at pamumuhunan ng Microsoft ay nagtatampok sa mas mataas na posibilidad ng pag-aampon mula sa mga cloud customers at development teams na naghahanap ng mga pagsulong sa produktibidad, habang ang interes ng enterprise—na naaakit sa pamamagitan ng mga alok mula sa Azure—ay maaaring pabilisin ang sukat. Sa parehong pagkakataon, ang hindi pa nalutas na mga pamamaraan sa pagsukat at performance na umaasa sa wika ay nangangahulugan na ang pag-aampon ay magiging hindi pantay-pantay sa pagitan ng mga proyekto at kumpanya, hindi man uniform o agad-agad.

Sa mas mahabang panahon, ang mga senaryo ay umaabot mula sa makabuluhang pagdaragdag ng mga human developers hanggang sa mas malawak na automation. Ipinakita ng CTO ng Microsoft na posibleng umabot ang generative AI sa 95 porsyento ng code generation sa 2030, ngunit ang hula na ito ay nakasalalay sa mga pag-unlad sa pagiging maaasahan, consensus sa pagsukat, at saklaw ng wika. Sa ngayon, ang estratehiya ng Microsoft ay muling bumubuo ng mga tool para sa developer at mga channel ng distribusyon ng enterprise habang iniiwan ang mga mahahalagang teknikal at evaluative na tanong na hindi pa nasasagot.
